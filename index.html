Architecture

OfOne compiles domain language into scenes, frames, tokens, moves, edges, loops, invariants, gates, criteria, tradeoff surfaces, and reviewable renderings. Adapters project domain semantics onto that geometry.

bounded objective + geometry primitives + adapter projection + scene map + evidence ledger + claim graph + causal graph + loop map + option moves + decision surface + update triggers + gates = decision rendering

Compiler Loop

The artifact is the source of truth. Prose is a rendering of validated state, and every implementation pass must be repairable, patchable, and measurable.

objective/context/sources -> draft artifact objects -> validate schema and semantic graph -> repair artifact failures -> render the smallest safe answer -> record patch triggers and benchmark trace

Artifact First

Claims, unknowns, criteria, gates, and recommendations must exist as addressable objects or explicit omissions.

Structured Diagnostics

Validator findings emit stable codes, severity, message, object metadata, path, and repair hints for CI, patching, and tests.

Recursive Review

Deep Research critique becomes typed sidecar state: inspected surfaces, accepted backlog, safety policy, convergence gate, and benchmark handoff.

Primitive Geometry

The system is a typed graph with loops. The visible chain is a traversal order, not a one-way checklist.

Scene

Bounded state-space snapshot: what exists, changes, and can be observed.

Subscene

Local scene for evidence acquisition, causal mechanism work, option decisions, proof steps, stakeholder context, or review gates.

Frame

Causal, logical, strategic, normative, temporal, or evidential coordinate system.

Token

Entity, variable, evidence item, claim, uncertainty, constraint, option, or trigger.

Unknown

Addressable absence: missing evidence, missing measurement, unresolved conflict, missing adapter, or unobserved variable.

Move

Infer, observe, test, intervene, decide, or revise.

Edge

Causal, evidential, argumentative, or workflow-state relation over cause, support, contradiction, constraint, update, block, or dependency.

Loop

Feedback, control, learning, incentive, measurement, contradiction, or review.

Decision Lifecycle

v0.6 keeps the core inquiry IR stable and hardens the recursive review protocol: identity, criteria, tradeoffs, actors, temporal validity, information value, lens coverage, council contention, review logs, recursive review cycles, convergence gates, benchmark traces, review sidecars, benchmark-case binding, pre-score auto-reject gates, excluded-run state, and immutable rerun semantics. Review-round labels such as v0.7 and v0.8 describe Deep Research cycles, not package or artifact versions.

Identity

Objective head, scope hash, config hash, active evidence hashes, created-at marker, and lifecycle status.

Criteria

Decision standards with priority, threshold, owner, and review implications.

Tradeoff Surface

Comparable options, dominant option, why it wins, and reversal conditions.

Actors

Decision owners, reviewers, affected parties, authority, incentives, exposures, and legitimacy basis.

Time

Decision horizon, deadline, evidence validity windows, staleness triggers, and update cadence.

Information Value

Which unknowns are worth resolving next, with impact, cost, time, and risk reduction.

Recursive Review

External critique becomes prepared, launched, researching, harvested, accepted, rejected, unresolved, implemented, and resubmitted state.

Standing Loop

A heartbeat may observe indefinitely, but each review cycle is bounded by harvest, adjudication, implementation, verification, publication, and next-mode decision.

Launch Proof

Deep Research handoffs require model, reasoning, context label, plan, Start, researching status, and stop-control evidence before they count as launched.

Research Watchdog

Active runs update ledgers only on material visible changes; ordinary unchanged polling makes no file changes, and stop-control blocks duplicate launches.

Status Isolation

Active external runs use run-scoped ledgers so launch, progress, harvest, and acceptance state cannot be confused across review cycles.

Convergence Gate

Architecture recursion stops or hands off to benchmarks when blockers are gone and remaining uncertainty is empirical.

Benchmark Trace

Benchmark readiness is explicit, and full-OfOne runs bind to case-file, prompt-file, and input-bundle hashes.

Case Binding

Full-OfOne benchmark artifacts must bind to the exact case, run, dossier, and prompt before any metric score can enter aggregate comparison.

Auto-Reject

Wrong-case artifacts, copied examples, required-output failures, leakage, or unsupported superiority claims are rejected before scoring.

Rerun Semantics

Excluded originals stay immutable; remedial reruns link back with explicit aggregate policy and cannot silently overwrite history.

Validation

OfOne artifacts are checked through JSON Schema profiles, then semantic graph rules for IDs, subscenes, unknowns, kill tests, references, relation legality, semantic relation families, adapter contracts, loop physics, decision surfaces, temporal validity, lens coverage, gates, review logs, rendering dependencies, and closure. Benchmark runs are checked separately for case fidelity, benchmark trace hashes, required outputs, independence, no-superiority compliance, machine-artifact hashes, excluded-run handling, rerun policy, and public checker attestations, because schema-valid is not benchmark-valid.
