Architecture

OfOne compiles domain language into scenes, frames, tokens, moves, edges, loops, invariants, and gates. Adapters project domain semantics onto that geometry.

bounded objective + geometry primitives + adapter projection + scene map + evidence ledger + claim graph + causal graph + loop map + option moves + update triggers + gates = decision rendering

Primitive Geometry

The system is a typed graph with loops. The visible chain is a traversal order, not a one-way checklist.

Scene

Bounded state-space snapshot: what exists, changes, and can be observed.

Subscene

Local scene for evidence acquisition, causal mechanism work, option decisions, proof steps, stakeholder context, or review gates.

Frame

Causal, logical, strategic, normative, temporal, or evidential coordinate system.

Token

Entity, variable, evidence item, claim, uncertainty, constraint, option, or trigger.

Unknown

Addressable absence: missing evidence, missing measurement, unresolved conflict, missing adapter, or unobserved variable.

Move

Infer, observe, test, intervene, decide, or revise.

Edge

Cause, constrain, support, contradict, enable, observe, evaluate, or update.

Loop

Feedback, control, learning, incentive, measurement, contradiction, or review.

Validation

OfOne artifacts are checked through JSON Schema profiles, then semantic graph rules for IDs, subscenes, unknowns, kill tests, references, relation legality, adapter contracts, loop physics, gates, rendering dependencies, and closure. Negative fixtures prove the validator rejects malformed maps.
